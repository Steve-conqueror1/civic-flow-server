# Spec for Department CRUD

branch: claude/api-feat/department-crud

## Summary

Implement full CRUD (Create, Read, Update, Delete) API endpoints for departments. The `departments` table and schema already exist. This feature adds the service, repository, controller, and router layers to expose department management over HTTP. Only admins and super admins may create, update, or delete departments; all authenticated users (and possibly public routes) may list and view departments.

## Functional Requirements

- **List departments**: Return all departments, with optional filtering by `isActive` status.
- **Get department by ID**: Return a single department by its UUID.
- **Get department by slug**: Return a single department by its slug.
- **Create department**: Accept `name`, `description`, `icon`, and `slug`. Auto-generate `slug` from `name` if not provided. Enforce uniqueness on `name` and `slug`.
- **Update department**: Allow partial update of `name`, `description`, `icon`, `slug`, and `isActive`. Re-validate uniqueness if `name` or `slug` is changed.
- **Delete department**: Soft-delete by setting `isActive = false`, or hard-delete â€” decide based on whether any services reference the department.
- **Role enforcement**: Create, update, and delete routes require `admin` or `super_admin` role via `requireRole` middleware.

## Possible edge cases (only if referenced)

- Attempting to create a department with a duplicate `name` or `slug` must return 409 Conflict.
- Attempting to get or update a non-existent department must return 404 Not Found.
- Slug generation from `name` must handle special characters, spaces, and Unicode safely (lowercase, hyphenated, ASCII-safe).
- Deleting a department that has associated services: decide whether to block the delete (return 409) or cascade/soft-delete.

## Acceptance Criteria

- `GET /api/v1/departments` returns a list of departments (200).
- `GET /api/v1/departments/:id` returns a department by UUID (200) or 404 if not found.
- `POST /api/v1/departments` creates a department (201), requires `admin`/`super_admin`, returns 400 on invalid input, 409 on duplicate name/slug.
- `PUT /api/v1/departments/:id` updates a department (200), requires `admin`/`super_admin`, returns 400 on invalid input, 404 if not found, 409 on duplicate name/slug.
- `DELETE /api/v1/departments/:id` deletes a department (200 or 204), requires `admin`/`super_admin`, returns 404 if not found.
- All validation errors follow the existing pattern: `{ errors: fieldErrors }` with HTTP 400.
- Router is mounted in `app.ts` under `/api/v1`.
- New Zod schemas live in `src/zodschemas/department.ts`.

## Testing Guidelines

Create a test file in the `./tests` folder for the new feature, and create meaningful tests for the following cases, without going too heavy:

- List departments returns all active departments.
- Get department by ID returns the correct record.
- Get department by ID returns 404 for unknown ID.
- Create department succeeds with valid payload.
- Create department returns 409 when name or slug already exists.
- Create department returns 400 when required fields are missing.
- Update department succeeds with valid partial payload.
- Update department returns 404 for unknown ID.
- Delete department returns 404 for unknown ID.
- Non-admin role is rejected (403) for create, update, and delete routes.
